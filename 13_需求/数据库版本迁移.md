#### 架构

多层架构设计。

1、接口层。
2、编译时生成层。
3、运行时核心层。

1、注解处理器层。
2、数据库持有层。
3、数据访问层。
4、执行层。

分层设计的好处：
1. 职责分明。
2. 数据类型安全。
3. 灵活性强。
4. 开发者只需关注 API 的调用和业务逻辑即可，门槛低。


#### 编译时注解处理器

是实现数据类型安全的核心。在编译时通过注解处理器，将所有带有Room注解的类生成对应的实现类。
