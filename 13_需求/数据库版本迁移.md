#### 分层架构

分层架构设计。

1、接口层。
2、编译时生成层。
3、运行时核心层。

1、注解处理器层。
2、数据库持有层。
3、数据访问层。
4、执行层。

分层设计的好处：
1. 职责分明。
2. 数据类型安全。
3. 灵活性强。
4. 开发者只需关注 API 的调用和业务逻辑即可，门槛低。

好处：关注点分离。

#### 编译时注解处理器

#### 自动数据变更通知

1、LiveData 集成。通过@Query 注解返回 LiveData 时，Room 会注册内容观察者。
2、RxJava 集成。通过 RxRoom 类提供 Flowable/Observable 支持。
3、数据变更检测。当数据库变更时自动触发通知。

好处：减少业务层对于数据手动刷新逻辑的编写。

#### 性能优化

1、编译时 SQL 验证。提前检查语句有效性，避免运行时错误。
2、语句缓存。缓存编译后 SQL 语句，减少重复解析开销。
3、批量操作。对批量插入/更新进行优化处理。
4、异步执行。通过@Transaction 注解支持异步事务。

好处：在保证类型安全的同时，提供良好的性能表现。
