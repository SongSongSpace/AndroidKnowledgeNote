**双层缓存**

- 内存缓存

- 快
- 杀进程就没

- 磁盘缓存

- 持久化
- 速度比内存慢
- 获取后，同步写入内存缓存池中。

- 目的：磁盘预热内存，核心是让慢磁盘的数据给快内存兜底，既保证持久化，又保证后续访问速度。

策略：读内存 -> 内存无/过期 -> 读磁盘 -> 磁盘无/过期 -> 查网络 -> 同时写内存 + 磁盘。

  

  

**三级缓存**（企业最标准的缓存架构）

- 内存 -> 磁盘/数据库 -> 网络

  

  

**HTTP** **缓存**

- OkHttp 自动帮做的

- 优先读本地缓存 -> 缓存过期才发网络请求
- 缓存直接存在磁盘（OkHttp自带缓存目录），不用自己管理文件。

- 是 OkHttp 基于 HTTP 协议自动实现的磁盘缓存。
- 缓存规则由 Cache-Control 头控制。
- 配置缓存目录。
- Retrofit 自动支持。
- 局限

- 只能按 URL 缓存整份数据，无法局部查询。
