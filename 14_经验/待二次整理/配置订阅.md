**Firebase Remote Config**

  

一、是什么？

  

是一个云服务客户端库，实现无需用户下载应用更新就可以更改应用的行为和外观，可以处理如提取和缓存参数值等重要任务，控制何时激活新的值。

  

二、Firebase应用于应用上。

  

前提条件：API19以上、Android4.4以上、使用Jetpack（build:grade 3.2.1以上、compileS DKVersion28以上）。依赖于Google Play服务的Firebase SDK 要求设备或模拟器上必须安装Google Play服务。

  

大致步骤：创建Firebase应用 ==》在Firebase上注册您的应用 ==》添加Firebase配置文件google-services.json ==》添加依赖项 ==》将Firebase SDK 添加到应用。

  

三、使用。

  

1、获取Remote Config对象。

2、设置应用内默认参数值。— 使用Map对象或存储在您应用的res/xml文件夹中的XML资源文件中指定一组参数名称和默认参数值。使用setDefaultsAsync()将这些值添加到Remote Config对象。

3、获取参数值。 — getBoolean() \ getDouble() \ getLong() \ getString().

4、后台设置参数值。

5、提取并激活值。 — 提取调用fetch()方法。参数值提供给应用调用activate()方法。 提取并激活调用fetchAndActivate()方法。谨慎选择激活值的时机。

6、实时监听更新。 —  使用addOnConfigUpdateListener()开始监听更新并自动提取任何新参数值。实现onUpdate()回调以激活更新的配置。

  

四、限制。

  

应用若在短时间内执行提取的次数太多，则提取调用会受到限制，并且SDK会返回FirebaseRemoteConfigFetchThrottledException。可以在应用临时设置一个具有较短最小提取间隔FirebaseRemoteConfigSettings.setMinimumFetchIntervalInSeconds(long)。

默认最小提取间隔为12小时。

**集成库**

  

1、初始化与Google Play连接

- 添加 Google Play 结算库依赖项
- 初始化 BillingClient

- BillingClient是Google Play结算苦与应用的其余部分之间进行通信的主接口。
- 创建 BillingClient，使用 newBuilder()，传递上下文。并设置setListener()传递purchaseUpdatedListener监听购买交易事件。
- 结算库会以BillingResult的形式返回错误，包含的BillingResponseCode会对遇到的错误进行分类。

- 连接到Google Play

- 调用startConnection()，连接过程异步。
- 必须实现BillingClientStateListener，以便在客户端的设置完成后且它准备好发出进一步请求时接收回调。
- 必须实现重试逻辑，以处理与Google Paly失去连接的问题。重写onBillingServiceDisconnected()回调方法，确保先连接，再发出进一步请求

  

2、展示可供购买的商品

- 先查询商品详情：

- 调用queryProductDetailsAsync()实现，并实现onProductDetailsResponse()，在查询完成时通知监听器。
- 查询时，传递QueryProductDetailsParams的实例。

- 使用Kotlin扩展程序进行查询

- 直接调用queryProductDetails()扩展查询，不用再定义单独的监听器。
- 会利用协程，该函数会挂起，直到查询完成。

- 处理查询结果

- 查询结果会存储在ProductDetails对象的List中。
- 在提供待售商品前，检查用户是否拥有该商品。
- 在提供订阅前，验证用户是否尚未订阅。

- queryProductDetailsAsync()会返回订阅商品详情，并每项订阅最多包含50订阅。
- 仅仅返回用户有资格享受的优惠。

  

3、启动购买流程

- 请从应用的主线程调用launchBillingFlow()方法。
- 上述方法接受对BillingFlowParams对象的引用。
- 方法会返回BillingClient.BillingResponseCode中的几个响应代码之一。当为OK时，则表示成功启动。
- 成功调用launchBillingFlow()后，系统会显示Google Play购买界面。
- Google Play会调用onPurchasesUpdated()。
- 必须实现onPurchasesUpdated()来处理可能的响应代码。
- 成功购买商品，系统会生成购买令牌。可以本地存储，建议传到安全的后端服务器。

  

4、指明个性化价格

- 若面对欧盟用户，使用setIsOfferPersonalized()方法向用户披露。

  

5、处理购买交易

- 大多会通过PurchasesUpdatedListener收到购买交易的通知。
- 但某些情况下，应用应通过调用BillingClient.queryPurchasesAsync()得知购买交易。
- 步骤：

- 验证购买交易。

- 一旦验证了交易，应用就可以向用户授权了。授权后，应用就必须确认购买交易，此确认会向Google Play告知您已经授予购买权。（三天时间确认购买交易。）
- 授予权利并确认购买交易：

- 确认购买交易前，资源的acknowledgementState为ACKNOWLEDGEMENT_STATE_PENDING。
- 消耗型商品：

- consumeAsync()方法：满足确认要求、表明授予权利以及让一次做商品可供再次购买。
- 表明一次性商品已经被消耗，请调用consumeAsync（），并添加应用商店在用户重新购买时提供的购买令牌。
- 必须传递一个实现ConsumeResponseListener接口的对象，该对象处理消耗操作的结果。可以替换onConsumeResponse()方法。

- 非消耗型商品：

- 使用BillingClient.acknowledgePurchase()或Product.Purchases.Acknowledge来确认购买交易。在此之前，应该使用isAcknowledge()方法或acknowledgementState字段检查购买交易是否已经确认过。

- 订阅：

- 与非消耗品类似，使用BillingClient.acknowledgePurchase()或Product.Purchases.Acknowledge来确认订阅。
- 所有初始订阅交易都需要确认，续订不需要确认。

- 向用户提供内容，并确认内容已经传送给用户。

  

6、提取购买交易

- 确保所有交易都可以被监听获取成功处理到，请确保应用在 onResume()方法中调用BillingClient.queryPurchasesAsync()。
- 提取交易记录：

- queryPurchasesAsync()会返回用户针对每个商品发起的最近的一笔购买记录，即使已过期、已取消或已消耗。
- 若使用Kotlin扩展，则可以使用queryPurchaseHistory()扩展函数。

  

7、处理在您的应用外进行的购买交易

- 发生在应用外，但希望应用显示应用内信息，或使用某种通知机制告知用户应用已经正确接收并处理该购买交易。

- 显示应用内弹出式消息。
- 将消息传送到应用内消息箱，并清楚地指明应用内消息箱中有新消息。
- 使用操作系统通知消息。

- 对后台发生的Play结算库操作使用指数退避算法，不影响用户体验。

  

**销售订阅内容**

  

1、处理订阅生命周期

- 检查订阅的状态：

- 使用Google Play结算库中的BillingClient.queryPurchasesAsync()。
- 使用Google Play API中的Purchases.subscriptionsv2: get()进行查询。
- 购买令牌的有效期是从订阅成功到订阅到期后60天。期限后，API 不可再用。

- 新订阅：

- 用户购买订阅后，BillingClient.queryPurchasesAsync()会返回订阅，并且系统会发送类型为SUBSCRIPTION_PURCHASED的SubcriptionNotification。
- 当收到通知时，应查询获取最新的订阅状态。然后再确认购买交易以及授予购买权利。

- 续订

- 续订不需要确认。
- BillingClient.queryPurchasesAsync()会返回订阅。
- 系统会发送SUBSCRIPTION_RENEWED通知。
- 应确保用户仍有权使用相应的订阅内容，然后使用返回的订阅资源中提供的expiryTime来更新订阅状态。

- 到期

- BillingClient.queryPurchasesAsync()不会返回订阅，用户失去对订阅的访问权限。
- 系统会发送类型为SUBSCRIPTION_EXPIRED的SubcriptionNotification通知。
- 当收到通知时，应查询获取最新的订阅状态。

- 取消

- 取消后，保有对相关内容的访问权限，直到当前结算周期结束为止。周期结束后，访问权限被撤消。
- 取消但尚未到期，BillingClient.queryPurchasesAsync()仍会返回订阅。
- 触发SUBSCRIPTION_CANCELED的SubcriptionNotification通知。
- 收到通知，会把subscriptionState设置为SUBSCRIPTION_STATE_CANCELED，并且expiryTime包含用户失去订阅访问权限的日期。若expiryTime是过去的日期，则马上失去订阅访问权限。否则，保留访问权限并直到过期。
- 可以从Google Play Developer API 返回的订阅资源中的cancelReason，了解原因。可以查看cancelSurveryResult字段，了解取消的原因。
- 应用需要一条消息，告知用户订阅已经被取消。

- 撤消

- 系统撤消用户的订阅：包括但不限于应用使用Purchases.subscriptions:revoke 撤消订阅或购买交易被退款。
- BillingClient.queryPurchasesAsync()不会返回订阅。
- 系统发送SUBSCRIPTION_REVOKED的通知。
- 收到通知时，会把从Google Play Developer API 返回的订阅资源中的subscriptionState设置为SUBSCRIPTION_STATE_EXPIRED，并且expiryTime包含用户失去订阅访问权限的日期。

- 宽限期

- 若应用仅依赖于queryPurchasesAsync()来检查用户是否有权访问订阅，则应该自动处理宽限期。
- 若应用与后端同步订阅状态，应该监听类型为SUBSCRIPTION_IN_GRACE_PERIOD的通知，以便用户在宽限期内收到通知。
- 宽限期时，订阅资源包含autoRenewEnabled = true。
- 应在应用内显示一条消息，告知用户如何修正付款方式。
- 修正了付款方式，订阅会续订。
- 未修正付款方式，会进入账号保留状态。
- 宽限期内，用户继续享受订阅权益。
- 若用户在宽限期内恢复订阅，续订日期不会重叠。
