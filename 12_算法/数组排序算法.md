## 冒泡算法
> [!基本思想]
> Bubble Sort is the simplest sorting algorithm that works by repeatedly swapping the adjacent elements if they are in wrong order.
> 
> 冒泡排序是最简单的排序算法，它可以通过重复交换相邻元素来进行排序。

难点问题：有几层循环？几层外循环？几层内循环？到底要循环多少次？
> [!难点问题]
> 外循环控制比较多少轮次。
> 内循环才是真正的比较数值进行交换。

数组共有 n 个元素。要比较多少轮次就能确定所有元素的位置呢？
- 答案：n - 1 次。
- 原因：当确定了 n-1 个元素的位置后，遗留的最后的一位的元素就已经确定了位置。
- 所以，比较时，外层循环循环 n-1 次即可。
```
# 第一次外循环
for i in range(n): # 网上这种写法很多，但其实是多余比较了最后一次
# 第一次外循环
for i in range(n - 1): # 由于比较这种东西 剩下一个肯定是要比较的 所以这样写才是对的
```

数组共有 n 个元素。要比较多少轮次的内循环能结束本次交换比较呢？
- 答案：= 已经排好序的 + 去掉自身。
- 每次比较完，最大的元素会被丢到最后。所以，每次内循环都要从下标 0 开始比较。

### 优化
#### 1、Early Exit
一次内循环的交换过程执行后，发现没有任何元素进行了交换，那么说明整个数组已经是排好序的。不需要在进行下一轮次的比较交换了。及早退出整个冒泡排序。
```
def bubble_sort_optimized(arr):
    n = len(arr)
    for i in range(n - 1):
      	# 每一次外循环的时候 写在里面
      	# 不要写在外面代码 每一次内循环之前要重置一下
        swapped = False
        for j in range(n - i - 1):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                swapped = True
        	# 如果没有交换 直接就跳出循环 说明整个数组已经是排序好的
        	# 要注意这个判断要写在第1个for里面 相当于第i次没有交换 整个程序目前都要break
          # 如果你写在了里面的for里面，你只能跳过里面的for
        if not swapped:
            break
    return arr
```
#### 2、Last Exchange Index
Smart boundary 不仅能发现数组是否已经有序了，还能明确数组后面有多少位已经排好序了。
```
def bubble_sort_smart_boundary(arr):
    n = len(arr)
    last_exchange_index = n - 1
    while last_exchange_index > 0:
        boundary = last_exchange_index
        last_exchange_index = 0 # 假设本轮没有交换 不写这个的话 while永远loop
        for j in range(boundary):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                last_exchange_index = j # 更新最后一次交换的位置
    return arr
```

我们用一个稍微有点“乱”但尾部有序的数组作为例子：`arr = [3, 4, 1, 5, 6]`。

初始状态

- `n = 5`
- `last_exchange_index = 4` (数组最后一个下标)

第一轮：`while` 第一次循环

此时 `boundary = 4`，我们要检查 `j` 从 `0` 到 `3`。

1. **j = 0**: 比较 `3` 和 `4`。 3 < 4，不交换。
2. **j = 1**: 比较 `4` 和 `1`。 4 > 1，**交换**！数组变为 `[3, 1, 4, 5, 6]`。
    - 关键点：`last_exchange_index` 更新为 `1`。
3. **j = 2**: 比较 `4` 和 `5`。 4 < 5，不交换。
4. **j = 3**: 比较 `5` 和 `6`。 5 < 6，不交换。

**第一轮结束结果**：

- 数组变为 `[3, 1, 4, 5, 6]`。
- `last_exchange_index` 停留在 **1**。
- **深层含义**：算法发现，下标 `1` 之后的所有元素（4, 5, 6）在这一轮都没动过。这意味着**下标 1 之后的所有东西已经全部归位了**！

第二轮：`while` 第二次循环

此时 `boundary = 1`（由上一轮的 `last_exchange_index` 赋值而来）。

这意味着 `for j in range(1)`，内层循环**只会运行一次**！

1. **j = 0**: 比较 `3` 和 `1`。 3 > 1，**交换**！数组变为 `[1, 3, 4, 5, 6]`。
    - 关键点：`last_exchange_index` 更新为 `0`。

**第二轮结束结果**：

- 数组变为 `[1, 3, 4, 5, 6]`。
- `last_exchange_index` 为 **0**。

结束：`while` 第三次尝试

- `while last_exchange_index > 0` 判定为 **False**。
- **跳出循环**，返回结果。
#### 3、CockTail Shaker Sort
与普通的冒泡排序不同。鸡尾排序是来回摆动排序。每次外循环都要确定一个最大值、一个最小值。本质还是通过对撞指针下的相邻比较。
```
def cocktail_sort(arr):
    n = len(arr)
    low, high = 0, n - 1
    while low < high:
        # 从左向右冒泡（找最大值）
        for j in range(low, high):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
        # 这里记得要写在全部排序找到最后在--，我第一次写在if里面了
        high -= 1 #找最大值 找到之后--

        # 从右向左冒泡（找最小值）
        for j in range(high, low, -1):
            if arr[j - 1] > arr[j]:
                arr[j - 1], arr[j] = arr[j], arr[j - 1]
        low += 1
    return arr
```

## 选择排序法
### 普通选择排序
> [!基本思想]
> 1、每次趟结束后，将待排序数组的最小值移动到最左边。即每趟找小值。
> 2、 双层循环。外层循环 0...n-1。内层循环 i+1..n-1。
> 3、每次内层循环结束，通过比较，确认未排序区间的最小值（最大值）。
> 4、外层循环结束后，排序结束。

时间复杂度：O(n^2)。
原地排序。常数空间。空间复杂度O(1)
```java
public int[] sortArray(int[] nums) {
    // 普通选择排序
    int n = nums.length;
    int min_index = 0;
    for (int i = 0; i < n - 1; i++) {
        min_index = i;
        for (int j = i + 1; j < n; j++) {
            if (nums[j] < nums[min_index]) {
                min_index = j;
            }
        }
        if (min_index != i) {
            int temp = nums[i];
            nums[i] = nums[min_index];
            nums[min_index] = temp;
        }
    }
    return nums;
}
```
### 进阶
#### 双向选择排序
每趟确定最小值和最大值。
注意：先确定的最小值，是否就是目标最大值。注意处理交换时的索引。

```java
public int[] sortArray(int[] nums) {
    // 双向选择排序
    int n = nums.length;
    int left = 0;
    int right = n - 1;
    int min_index = 0;
    int max_index = 0;
    while (left < right) {
        min_index = left;
        max_index = right;
        for (int i = left; i < right + 1; i++) {
            if (nums[i] < nums[min_index]) {
                min_index = i;
            }
        if (nums[i] > nums[max_index]) {
                max_index = i;
            }
        }

        int temp = nums[left];
        nums[left] = nums[min_index];
        nums[min_index] = temp;

        if (max_index == left) {
            max_index = min_index;
        }

        temp = nums[right];
        nums[right] = nums[max_index];
        nums[max_index] = temp;

        left++;
        right--;
    }
    return nums;        
}
```
#### 堆排序

> [!基本思想]
> 1、建堆。
> 2、将堆顶元素与堆尾元素交换。交换后，再递归堆化建堆。让堆符合堆性质。
> 3、 开始循环第二点，直到所有元素都交换完成。
> 4、最后得到的数组即为堆按层排序的结果。

```java
class Solution {
	public int[] sortArray(int[] nums) {
		// 堆排序
		// 1、建堆，从最后一个非叶子节点开始。节点数 / 2 - 1。
		// 最大堆
		int n = nums.length;
		
		for (int i = n / 2 - 1; i > -1; i--) {
			heapify(nums, n, i);
		}
	
		for (int i = n - 1; i > 0; i--) {
			// 交换堆顶和当前末尾
			int temp = nums[i];
			nums[i] = nums[0];
			nums[0] = temp;
			// 堆缩小后的堆（大小 i，从堆顶下沉）
			heapify(nums, i, 0);
		}
		return nums;
	}
  
	public int[] heapify(int[] nums, int n, int i) {
		int largest = i;
		// 节点的左节点索引等于节点索引 * 2 + 1
		// 节点的右节点索引等于节点索引 * 2 + 2
		int left = 2 * i + 1;
		int right = 2 * i + 2;
		if (left < n && nums[left] > nums[largest]) {
			largest = left;
		}
	
		if (right < n && nums[right] > nums[largest]) {
			largest = right;
		}
	
		if (largest != i) {
			int temp = nums[i];
			nums[i] = nums[largest];
			nums[largest] = temp;
			heapify(nums, n, largest);
		}
		return nums;
	}
}
```
找最值。建堆是个好方法。可以将时间复杂度从O(n)降低到O(logn)。
即时间复杂度：O(nlogn)。
原地排序。
不稳定。
##  插入排序法

> [!NOTE] 思想
> 1、将给定数组在逻辑上看做两部分。一部分是有序数组，一部分是待排序数组。有序范围从最左边索引为 0 的元素开始，一直向右扩张。
> 2、每趟从待排序数组取最左边一个元素，将其遍历的与有序数组元素比较，从有序数组的右边开始比较，如果待排序元素小于当前元素，则将当前元素向后移动一位。否则，就将待排序元素插入在比较元素之后。

可以通过嵌套两层for循环。也可外层for循环，内层while循环。
示例代码如下：
```java
public int[] sortArray(int[] nums) {
	// 插入排序
	int n = nums.length;
	if (n <= 1) return nums;
	for (int i = 1; i < n; i ++) {
		int temp = nums[i];
		int j;
		// j = i - 1;
		// while (j >= 0 && nums[j] > temp){
			// nums[j + 1] = nums[j];
			// j--;
		// }
		for (j = i - 1; j >= 0 && nums[j] > temp; j--) {
			nums[j + 1] = nums[j];
		}
		nums[j + 1] = temp;
	}
	return nums;
}
```
时间复杂度：O(n^2)。
空间复杂度：原地排序O(1)。
稳定排序。
适合规模小的数组、基本排序的数组。
### 进阶
#### 希尔排序

> [!NOTE] 基本思想
> 1、通过设定不同的间隔（gap），将数组**分组**进行插入排序，然后逐步缩小间隔直至为 1，最终完成整个数组的排序。
> 详细步骤：
> 1、 设定初始间隔 gap = n / 2。
> 2、按间隔将数组分组，对每组进行插入排序。
> 3、缩小间隔，gap = gap / 2。
> 4、重复步骤 2～3，直至 gap = 1。
> 5、最后对整个数组进行一次插入排序。

| 指标          | 复杂度               | 说明                  |
| ----------- | ----------------- | ------------------- |
| **最佳时间复杂度** | O(n)              | 当数组已有序时             |
| **最坏时间复杂度** | O(n^2)            | 使用普通间隔序列时           |
| **平均时间复杂度** | O(n^1.3)~O(n^1.5) | 取决于间隔序列选择，如果选取得当接近于 |
| **空间复杂度**   | O(1)              | 原地排序，只使用常数空间        |
| **稳定性**     | 不稳定               | 不同组间的相等元素可能改变相对顺序   |
```java
public int[] sortArray(int[] nums) {
	// 希尔排序 shell sort
	int n = nums.length;
	int gap = n / 2;
	  
	while (gap > 0) {
		for (int i = gap; i < n; i++) {
			int temp = nums[i];
			int j = i;
			while (j >= gap && nums[j-gap] > temp) {
				nums[j] = nums[j-gap];
				j = j - gap;
			}
			nums[j] = temp;
		}
		gap = gap / 2;
	}
	return nums;
}
```

不稳定。
适合中等规模数组排序。
